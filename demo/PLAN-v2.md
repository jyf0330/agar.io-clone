## 一、为什么有 V2：V1 的 5 个致命弱点（背景，新同事看）

### 项目风险（前 5 条）

1. **LLM 点评强依赖外网**：demo 现场可能没网 / API 被墙 / key 失效。一旦失败，90 秒视频的核弹镜头直接哑炮。
2. **"4 人同 WiFi 联机"也是坑**：谁跑 server、防火墙、IP 发现。现场 10 分钟就能把你磨到崩溃。
3. **Day 1 就踩画板 Bug**：master 分支那个"画不出来"的根因未定位，可能卡 2 天。整个计划塌方。
4. **3 分钟局 × 90 秒 demo**：要么剪辑加速（看起来假）、要么跳切（叙事断）。节奏天然不对。
5. **T 台 12 秒 + LLM 打字机**：时间给不够，观众没看完就切。

### 玩家 / 评委抗拒（后 5 条）

6. **"空白画板"门槛**：90% 的人面对白纸 30 秒画不出像样的东西，开局就尴尬。
7. **"画丑被展示"社交焦虑**：不是所有人都愿意把自己的画作公开给评委看。
8. **"被偷部位"单向挫败**：你精心画的眼睛被别人抢走 = 纯负反馈，没有反手快感。
9. **"T 台强制停战"挫败**：打正 high 被按住播片尾，主动权被夺。
10. **LLM 扎心**："P3 像被母亲遗弃的婴儿"——真有可能生成这种东西，评委坐不住。

## 二、V2 五大核心改动（方向已锁）

| # | V1 原设计 | V2 改法 | 解决的风险 |
|---|---|---|---|
| 1 | 空白画板，玩家从零画 avatar | **5 张 AI 生成的残缺骨架随机选一张，玩家只"涂色 + 加笔" 20 秒** | #3 #6 #7 |
| 2 | 真实 4 人同 WiFi 联机 | **2 个真人本地同屏 + 2 个预录 Bot 回放**（服务端把 bot.json 当玩家喂进去） | #2 |
| 3 | 局末强制 T 台停战 30 秒 | **胜利触发瞬间进入"慢动作回放 + 逐个人物亮相"**（自然结束，不是强制打断） | #9 |
| 4 | 死亡时"抢部位"替换对方那一层 | **死亡时用你的颜料在对方那一层上"画一笔"**（30% 透明度覆盖，不删除对方原作） | #8 |
| 5 | 局末 LLM API 实时生成点评 | **本地 50 条诡异点评池 + 按特征查表匹配**；LLM 做 async bonus（成功就盖，失败就本地） | #1 |

### 三个附加小改

- **局时 180s → 90s**：demo 视频 = 完整一局 + 10 秒回放。零剪辑。
- **胜利条件改为**：`剩余原创部位数 × 2 + 对他人成功涂笔数`——鼓励双向交互，不是单向抢。
- **LLM prompt 强制约束**：语气荒诞但温柔，禁止涉及遗弃/死亡/母亲/父亲/贬低，只描绘图像意象。硬规则 + fallback 双保险。

### Day 3：涂改机制（不抢，只叠）

- 死亡判定触发时：受害者 avatar 的中央位置 **+ 一道 30% 透明的 brush stroke**（颜色 = 攻击者主色调）。
- 视觉 toast：**"P1 在 P2 身上画了一笔"**（不是"抢走"）。
- 记分：
  - 攻击者 `涂笔数 +1`
  - 受害者 `原创度 -0`（**不扣**，只是被涂）
  - 胜利公式：`剩余原创部位数 × 2 + 对他人成功涂笔数`
- 技术上只叠 PNG 不改 JSON 层级。

**✅ Day 3 验收（19:00）**

## 四、V2 额外风险提示（执行期要留意）

1. **"只涂不抢"会让部分玩家觉得没有惩罚感**——这是审美权衡，demo 阶段优先保"可爱"情绪，正式版再考虑加惩罚。
2. **本地点评池需要策划亲自审稿**——程序员写不出氛围文案，策划必须亲手过 50 条。Day 4 晚上 21:00 前完成。
3. **Bot 回放看起来"像真人"需要一点调优**——Bot 动得太机械会露馅。轨迹里我已经加了"fake chase"片段（每 10 秒冲中心 2 秒），但如果 Day 2 验收发现还是机械，调 `generate_bots.py` 的 `DEFAULT_JITTER` / `DEFAULT_TARGET_SWAP_SEC` 再跑。
4. **demo 场地如果有投影/大屏**——Canvas 的 320 像素图放到 4K 大屏会糊。Day 6 美化阶段就要测一次，不要留到 Day 7。
5. **LLM prompt 要跟合作方再过一遍**——如果 LLM 接口有自带的"安全策略"会 re-write 你的输出，需要在 demo 前一天实测，不然 Day 5 的 bonus 变惊吓。
