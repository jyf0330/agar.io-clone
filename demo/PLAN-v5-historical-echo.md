## 一、一句话

2–4 人联网 PvP，单局 **30–60 分钟**；玩家操控 **带部位的固定人形角色**（非纯圆球抽象）；地图 **跨局固定坐标**（或可复现 seed，与历史一致）；**真实玩家历史会话**在 **时间与地点同时命中** 时在世界上回放为 **鬼魂**（可多条、不绑定玩家）；鬼魂 **穿体无碰撞**、**全员可见聊天气泡**；跟丢则消散。场上有 **1 只跟宠 + 地图上多只可切换宠物 NPC（AI 层）**：**LLM + 跨局记忆**，引导 **部位 / 热点 / 回声**；**跟宠**记住与玩家的共同事实，对话可 **跨局引用**（须 L1 可校验）。开局 **固定角色立绘** 展示 **缺一格**，玩家在该格 **涂鸦**；与参考图 **感知哈希相似度**：**&lt;10%** → 不用玩家图、按参考图默认出战、**无签名加成**；**≥10%** → 签名生效 + **基础加成**（如移速 +10）；**≥70%** → 在基础上 **+50% 该条加成**（如再 +5，合计 +15，数值可配）。部位 **槽位换装**，旧件 **掉地**；掠夺须 **完整部位对象**。部位来源 **四条**：击杀、地图拾取、鬼魂绝对坐标、**地图 NPC 任务**。**胜利**：**时间到** 或 **有人凑齐当局定义的全部部位**。**结局全身拼图式合成展示待定**。

## 二、双层结构：鬼魂（事实）× 宠物 NPC（AI）

| 层 | 职责 | 技术 |
|---|---|---|
| **鬼魂** | 过去真人在这张图上 **真实发生过什么**（轨迹、聊天、拾取点） | 会话录制 + 确定性回放；**不**靠 LLM 编造行为 |
| **宠物 NPC** | **陪你理解世界、记住你、带你去挖宝**；把「回声」和「你这局目标」缝进对话里 | **LLM**（[`wrapper`](../apps/server/src/llm/wrapper.js) + orchestrator 思路）+ **SQLite 记忆**（事件 / 局末摘要 / 可选人物印象） |

**为什么要两层**：赛方 **Track B** 要求 **长期记忆、人设、深度互动**——单靠录像回放不够；宠物承担 **AI 赛道叙事**。鬼魂承担 **怪诞感与真实历史**。玩家体验是：**宠物像能穿越时间的向导**，而 **鬼魂是钉死在过去的那一截真实**。

## 三、宠物 NPC（1 跟宠 + 地图多只可切换 · 寻物 · 记忆 · 跨时空对话）

### 3.1 数量与身份

- **跟宠**：任意时刻 **恰好 1 只** 跟随玩家，承担主对话与主引导；独立 `npcId`、人设卡（可沿用 [`demo/npcs/personality-cards/`](../demo/npcs/personality-cards/) YAML）。
- **地图宠物**：同图 **多只** NPC 宠物游荡或可交互；玩家可 **切换跟宠**（换绑 `activePetId`），切换后 LLM 上下文与记忆键仍为 **(playerId, npcId)** 按宠物分别持久化。
- **非**鬼魂；**实时 AI** 驱动实体（碰撞/吞噬规则：**默认** 可推开、不可被吞，避免 PvP 崩坏）。

### 3.2 带玩家找东西

- **输入**：服务器侧已知或推断的 **热点**（例如「未来 Δt 内可能在 (x,y) 附近触发某条回声」「地图散落部位刷新点」「历史 `item_events` 密集区」）。
- **输出**：宠物 `move_to` / 「跟我来」类意图，向 **绑定或最近的玩家** 靠近并领头移动（实现上对齐现有 `Orchestrator` 的 intent：`move_to` / `speak`，可加 `hint_loot` 事件由服务端下发）。
- **v1 可简化**：热点来自 **规则表 + 下一条将触发的鬼魂坐标**（不必每帧 LLM 策展）；LLM 负责 **怎么说**，不负责 **算坐标**。

### 3.3 同一只宠物记住什么（跨局）

对 **(playerId, npcId)** 持久化，至少：

- **L1 事件**：`guided_to_echo`、`gifted_part`、`player_picked_with_me`、`chat_turn` 等（**只存事实**，供审计与摘要）。
- **L2 局末摘要**（每局结束 1 次 LLM 或模板）：「这局带 ta 去了东南回声 / 给了 ta 一只手」。
- **L3 人物印象**（每 N 局压缩）：短文本 **「ta 爱跟鬼魂抢东西 / 总问我心脏在哪」**，**注入下次对话 system prompt**。

**同一只宠物**下次见同一玩家时，台词可引用：**「上次我们找到的那只手，你还留着吗？」**——即 **跨越时空的对话**（时间指真实日历 / 局序号，不是游戏内 30 分钟一局内的伪时间）。

### 3.4 与玩家聊天

- 玩家打字 → `npc_reply` 批量或单条；注入 **人设锚点 + 最新 L2/L3 + 本局摘要**（token 预算同 Track2 约束）。
- **禁止**：编造不存在的拾取记录（幻觉）→ **事实校验**：涉及「给过你什么」必须能 `grep` 到 L1 事件 ID 或拒绝回答。

### 3.5 与现有代码复用

- [`apps/server/src/npc/orchestrator.js`](../apps/server/src/npc/orchestrator.js)、[`prompts.js`](../apps/server/src/npc/prompts.js)：**逻辑复用**，扩展 context（热点坐标、鬼魂即将出现窗口、玩家当前部位列表）。
- 记忆表：可沿用 Track2 PLAN 中 `events` / `session_summaries` / `persona_impressions` 设计，挂载到 **V5 的 playerId + petId**。

## 四、鬼魂与回放

| 项 | 定案 |
|---|---|
| 鬼魂来源 | **单条历史会话 = 单条鬼魂**（不做多人跨 session 语义缝合） |
| 出现规则 **v1** | **时间 + 地点双匹配**：当前局时钟落在历史事件 `t ± 容忍窗` 且 **历史绝对坐标** 距 **任意当前玩家** ≤ 触发半径 → 在该坐标开始/继续回放 |
| 相对路径物品 | **不做**；拾取物在 **历史记录的绝对坐标**、对应 `t` 生成 |
| 地图 | **固定**或与历史 **一致** 的可复现布局 |
| 同局多条鬼魂 | 允许；每条独立；不与单个玩家强制绑定 |
| 跟丢 | 半径 **R**、连续 **T** 秒无真人进入范围 → 消散（参数可调，曾讨论 R≈1500、T≈30） |
| 时间轴 | **1:1**；历史会话结束或当前局结束 → 鬼魂结束 |
| 碰撞 / 攻击 | 鬼魂不可被攻击；与当前玩家无碰撞 |
| 聊天 | **实时**气泡于鬼魂头顶；**全图玩家可见** |
| 录制 | 会话写入 DB；**录制同意** 开关（不同意则不进入公共回放池） |
| 冷启动 | **5–10 条** 开发者种子会话；首批玩家提示「在给后人留影子」 |
| LLM 策展（仅鬼魂池） | **v1 可不做**；用规则 + 种子库挑回放。 **宠物 NPC 的 LLM 对话与记忆为必做**（赛道 B） |
| 回放采样 | **200ms** 位置采样 + **线性插值**；**不**回放 split 多球（鬼魂单体） |
| 穿墙 / 出界 | 不特殊处理（幽灵合理） |

## 五、部位与装备

### 5.1 开局 · 固定角色缺一格 · 哈希分档

- **UI**：**固定角色全身像**（或等价立绘），除 **签名槽** 外均已装配；签名槽类型与当期 **三张参考图** 一致。
- **流程**：玩家涂鸦 → 与 **最佳匹配参考图** 算相似度 `s`（由哈希距离映射到 0–100% 或等价档位，阈值可配）：
  - **`s` &lt; 10%**：丢弃玩家笔迹，槽位使用 **参考图默认外观**，`templateId` 锁定为该参考，`signatureBonus = 0`。
  - **`s` ≥ 10%**：采用玩家签名（渲染可叠笔迹或色调映射），`templateId` 同上，应用 **基础加成**（例：移动速度 +10，按配置表）。
  - **`s` ≥ 70%**：在基础加成上 **×1.5（+50% 该条）**（例：+10 → 再 +5，合计 +15）。
- **实现候选**：`simon987/imagehash-web`、`browser-image-hash`；**阈值与加成表** 放 [`configs/game/`](../configs/game/) 或 V5 专用配置。

### 5.2 槽位与换装

- 开局只带玩家签名生成的那一件部位；后续获得的完整 part 对象会进入本局部位清单，不再把旧件挤掉。
- 获得同类型部位时保留多件副本，属性按现有 body bonus 规则汇总，局内清单展示来源和属性。
- **部位来源（四条）**：**击杀掠夺**、**地图散落拾取**、**鬼魂绝对坐标与时间刷新**、**地图 NPC / 任务奖励**（含宠物任务或其他任务 NPC）。
- **掠夺、鬼魂、地图、任务** 均须传递 **完整部位对象**（`type`、`templateId`、`role`、`userStroke` 可选、溯源字段可选）；禁止仅 `{ type }`。

### 5.3 胜利条件

- **局时结束**（达到配置时长）；或  
- **任一玩家凑齐当局定义的核心部位集合**，且除了玩家签名生成的那一件以外，其余核心部位都来自外部来源。

### 5.4 与现有代码

- [`apps/server/src/body.js`](../apps/server/src/body.js) 中 `stealRandomCorePart` 当前只追加 `type`，**须升级为** 完整对象 + 槽位与掉落。
- `materialization`：v1 可先 **HUD 阶段切换**；深度规则后续接 [`docs/10-custom-systems.md`](../docs/10-custom-systems.md)。
- **表现层**：客户端由「纯圆球」扩展为 **人形/立绘 + 部位挂点**（与现 player-card / avatar 管线衔接，实现待定）。

## 八、风险摘要

- 地图必须与历史坐标一致。
- 双匹配过严 → 可选 HUD 弱提示「历史热点」方向。
- SQLite 原生在部分 macOS 环境失败 → 优先 `better-sqlite3` 等。
- 换装与同步协议改动面大 → 服务端权威。

## 九、评委叙事（Track B · 智能 NPC）

- **长期记忆**：宠物 **L1→L2→L3**；鬼魂 **会话库**（事实锚）。演示：关游戏 5 分钟再开，宠物第一句引用上局。
- **人设一致性**：人设卡 anchors 强制注入 + 标准问句回归测试（可沿用 Track2 一致性脚本思路）。
- **记忆膨胀 / 幻觉**：原始事件不进长 prompt；只注最新摘要 + L3；赠与 / 寻物 **必须对 L1 可校验**。
- **深度互动**：玩家聊天 + 指令跟随（「带我去有回声的地方」）；宠物结合热点 `move_to`。
- **鬼魂**：增强 **差异化体验**（真实历史回声），**不替代**宠物的 AI 记忆义务。
- **离线**：鬼魂回放不依赖 LLM；宠物走 fallback 池 + 缓存（[`wrapper`](../apps/server/src/llm/wrapper.js) 既有策略）。
