## 方向提案（Demo 之后再认真评估，V3 主线不受影响）

### `proposal-V4-organ-symbiosis` · 器官拼接 + 共生 + 背叛

**提交日**：2026-04-24
**来源**：用户贴的"顶尖策划三层抽象" pitch
**当前状态**：**冻结在 icebox**。V3 赛道 2 继续执行，21 天内**不讨论**、**不原型**、**不在代码里试**。V3 提交后正式评审。

**一句话核心（pitch 原文）**：
> 一个基于"器官拼接"的多人同场单人博弈游戏，玩家在漂浮环境中通过**临时共生、随时背叛**，争夺资源并最终拼出完整身体成为唯一胜者。

**关键词**：共生 ≠ 组队；拼接 ≠ 收集；背叛是核心玩法，不是附加。

**三层结构**：

- **核心系统层**（游戏成立的最小闭环）
  1. 共生系统 — 物理绑定，通过"躯干"触发连接；共享位置（强绑定）+ 技能组合效果（弱协同）；可随时断开 → **背叛合法化**。本质：不稳定关系系统。
  2. 拼接系统 — 所有行为围绕"拼出完整身体"；必须先拿躯干，其他器官必须挂载；拼接同时是资源占有 / 权力象征 / 被围猎目标。本质：从"收集"升级为**结构构建竞争**。
  3. 能量系统 — 孢子 = 能量来源；多吃 → 强但暴露，少吃 → 弱但安全；能量 → 大招 → 改变战局。本质：风险可视化成长（类似吃鸡开枪暴露）。

- **策略层**（让游戏有深度）
  1. 信息不对称 — 不同器官 = 不同信息权（👁眼睛全局 / 👂耳朵隐藏 / 🦶脚掌位置规避 / ❤️心脏控场干扰）。结果：比"信息利用"，不比操作。
  2. 关系博弈 — 不是队友也不是敌人，而是**暂时互利体**（合作抢资源 / 半路断开 / 利用对方技能 / 背刺抢身体）。类比：Agar.io + Among Us 心理博弈层。
  3. 风险-收益博弈 — 吃孢子（变强 / 被发现）、合体（更强 / 被背叛）、收集器官（接近胜利 / 成为目标）。**始终没有安全选项**。

- **体验层**（项目最有杀伤力的地方）
  1. 情绪曲线 — 前期孤独+探索 / 中期信任+利用 / 后期猜疑+背叛 / 终局猎杀 or 被猎杀。**情绪驱动**而非纯机制。
  2. 视觉表达服务玩法 — 柔软环境 + 残缺身体 + 动态拼接 = 不稳定生命体（关键不是"子宫"，而是漂浮 / 柔软 / 拼接三要素）。
  3. 可读性设计 — 发光 = 强、连接线 = 关系、器官形态 = 能力、分离瞬间 = 背叛信号。观众一眼看懂谁要死。

- **节奏层**（三阶段 + 一条核心规则）
  - 发育期（低冲突）→ 共生期（高变数）→ 狩猎期（高对抗）
  - **节奏不是时间推进，而是"身体完整度"推进**。

**最终升维总结**：
> 围绕"身体完整性"的权力博弈系统：玩家通过建立与破坏临时共生关系，在信息不对称与风险暴露中竞争，最终以"拼接完成的身体"作为唯一胜利条件。

**设计者自己的一句点评（重要）**：
> 真正牛的点不是"器官 / 拼接 / 子宫场景"，而是**"关系不稳定 + 身体绑定"的博弈机制**。下一步重点："把背叛瞬间做成最爽点"（突然断开 + 抢走器官 / 共生时反向释放技能 / 假合作骗技能 CD）。

**和当前 V3 的关系**：

| 维度 | V3 赛道 2（当前主线） | V4 pitch |
|---|---|---|
| 玩法 | 单人 + 3 AI 宠物花园 | 多人同场 + 物理共生 + 背叛 |
| 核心卖点 | 长期记忆 + 人设一致性 + 情感纽带 | 不稳定关系 + 身体绑定博弈 |
| 赛道适配 | 腾讯 AI NPC 赛道 2 | 可能的 Indie 玩法创意赛道 / 别的赛事 |
| 技术复用 | wrapper.js / cache.js / audit.js / personality-loader | **几乎全部不复用**（多人对抗重做） |
| 核心代价 | 已落地 Week 1 milestone | 21 天归零 |

**为什么先冻结而不是立刻讨论**：
1. V3 方案锁定声明（PLAN-v3 §九）已于 2026-04-24 生效，距今不足 24 小时。
2. 触发 PLAN-v3 §八 R10（"2 周后你没信心了想回 V2"）—— 这次是 1 天内换方向，正是 R10 要防的场景。
3. pitch 写得漂亮但**没原型、没 playtest、没核心机制可行性验证**；V3 已有 4 份 milestone 录像 + 已实现 wrapper/npc 代码。
4. 评估 V4 可行性本身要花 ≥ 2 小时（机制博弈平衡分析 / 小型原型 / 竞品对比），不是能顺手做完的事。

**解冻条件**（任一满足时重新评估，否则 Demo 日前不讨论）：
- V3 提交完成（Day 21 之后）。
- V3 在 Day 7 / 14 里程碑**连续两次**过不了、且按 PLAN-v3 §四砍功能顺序砍完仍不达标——这时才是"换方向"的合理触发器，不是现在。

**禁止行为（21 天内）**：
- 不在 V3 代码里为 pitch 留"扩展点 / 插件位"。
- 不把 pitch 概念（共生 / 背叛 / 器官拼接）混进 V3 的 NPC 行为设计。
- 不和策划 / 同事口头讨论"要不要切 V4"——讨论信号本身就会动摇 21 天执行。
